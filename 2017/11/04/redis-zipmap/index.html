redis zipmap

zipmap介绍

redis中，zipmap用于存储小型的字符串映射关系，能够高效的利用内存空间。

假设zipmap中存储的字符串映射为 foo => bar 和 hello => world，则对应的zipmap结构如下图所示：

其中zmlen的长度为1字节，记录了zipmap所保存的键值对的数量，当此值大于或者等于254时，需要遍历整个zipmap才知道所有键值对的数量。

len记录其后面字符串的长度，len可以使用1字节或者5字节来编码，如果len第一字节为253，表示此字节之后的4字节无符号整数存储的是字符串的长度，如果len第一字节介于0~252之间，此值即为字符串的长度。值254用于标记未使用的，可以添加键值对的空间。

free为uint8_t，记录空余的空间的长度。

ZIPMAP_END标记的是zipmap的结束位置，其值为255。
